pip install playwright markdownify readability-lxml
3 packages instead of Firecrawl's 5-service architecture. Playwright for browser, markdownify for HTML-to-Markdown, readability for content extraction.
5:00 — Core Engine
scraper.py — 400 lines, Firecrawl-compatible API
Full implementation: async Playwright browser, lazy init, HTML-to-Markdown, Firecrawl-compatible actions (click/write/wait/scroll/screenshot), file cache (2 days), batch parallel scraping.
8:00 — AI Extraction
Free AI JSON extraction
Instead of paying for Firecrawl's LLM extraction ($16+/mo), plug in any free LLM API. Send page content + prompt, get structured JSON back. $0 cost.
10:00 — MCP Server
JSON-RPC 2.0 MCP server for Claude Code
3 tools: firescrape_scrape, firescrape_batch, firescrape_extract. Drop-in replacement, registered in Claude and working.
12:00 — Testing & Tuning
Bug fixes: readability fallback, timeout optimization
Fixed browser reuse across asyncio.run() calls, added smart content fallback (readability too aggressive? use full HTML), reduced networkidle timeout from 10s to 5s.
15:00 — Benchmark
Full comparison: 5 URLs, 4 AI models analyzing results
100% success rate on both tools. Orchestrated team test: FireScrape + 3 AI models vs Firecrawl + 1 AI model. Results below.

Verdict

Firecrawl is 11x faster at raw scraping (cloud CDN vs local browser).
But that speed advantage is irrelevant.

Why Scraping Speed Doesn't Matter

Firecrawl scrapes in 0.5s vs FireScrape's 5.6s. Sounds like a huge win. But look at the full pipeline:

Firecrawl's total time: 16.8s. Scraping was only 3% of that. The other 97% is waiting for AI to analyze the content.

Scraping speed is Firecrawl's marketing pitch. In reality, it's the least important part of the pipeline.

Why 3 Models Beat 1

FireScrape sends the same content to 3 AI models in parallel. They all run at the same time, so the total wait = slowest model (14.5s), not 3x the time.

Result: 3 independent analyses with different perspectives, styles, and insights. One model catches what another misses. All 3 found 12/12 articles — but each highlighted different themes.
